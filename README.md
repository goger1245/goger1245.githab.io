# Portfolio Website

**Сайт‑визитка / портфолио.** Этот репозиторий нужен, чтобы:
- показать **уровень разработки** (структура, качество кода, аккуратность);
- дать единый публичный источник информации об условиях взаимодействия;
- оставить понятный пример того, как я оформляю фронтенд‑проекты.

---

## Условия и формат работы

- **Коммуникация**: фиксируем задачу, требования, ограничения и критерии “готово”.
- **Оценка**: сроки/стоимость + риски и зависимости (что может повлиять на план).
- **Реализация**: итерациями, с промежуточными демо и прозрачным прогрессом.
- **Сдача**: передача результата + помощь с внедрением/деплоем (по необходимости).
- **Поддержка**: оговаривается отдельно (фиксы/правки/развитие).

## Подход к качеству

- **Читаемость**: понятные имена, короткие функции, минимум “магии”.
- **Поддерживаемость**: логика разнесена по слоям, нет дублирования.
- **Предсказуемость**: единый стиль форматирования и соглашения по структуре.
- **Аккуратность**: чистая история без мусорных файлов и случайных артефактов.

## Структура сайта

```
index.html          # единая HTML‑точка входа
src/
  assets/           # изображения/ассеты
  styles/
    main.css        # стили
  js/
    main.js         # bootstrap / wiring
    nodes.js        # основной контроллер интерфейса
    i18n.js         # локализация
    animation.js    # интро/анимации
    utils.js        # утилиты
    components/     # переиспользуемые части (canvas/геометрия/анимации)
    system/         # модули интерфейса (например: hover/reveal)
```

## Быстрый старт (локально)

Проект статический — сборщик не требуется.

- Открой `index.html` через любой локальный сервер (например, Live Server).

## Инструменты разработки (опционально)

В репозитории лежат конфиги для единых стандартов:
- **EditorConfig**: `.editorconfig`
- **Prettier**: `.prettierrc.json`, `.prettierignore`
- **ESLint**: `eslint.config.js`

Если используешь Node.js:

```bash
npm i
npm run format
npm run lint
```
